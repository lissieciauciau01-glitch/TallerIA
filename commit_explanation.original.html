¿Qué es un commit y por qué es importante?

Un commit es una captura del estado del proyecto en un sistema de control de versiones (por ejemplo Git). Guarda cambios, mensaje y metadatos (autor, fecha, hash).

Por qué es importante
  • Historial: registra qué cambió, cuándo y por qué.
  • Reversión: permite volver a estados anteriores fácilmente.
  • Colaboración: otros pueden revisar y combinar cambios.
  • Trazabilidad: vincula cambios a issues y despliegues.
  • Buenas prácticas: facilita branches, PRs y bisect.
Buenas prácticas rápidas
  • Commits pequeños y atómicos (una idea por commit).
  • Mensajes claros en modo imperativo: "Añade validación en suma".
  • Probar antes de commitear.
  • No incluir secretos ni archivos generados (usar .gitignore).
Comandos básicos 
git add .                      # añade cambios al área
git commit -m "Mensaje claro"   # crea un commit
git commit -am "Mensaje"        # añade y commitea archivos ya trackeados
git log                         # ver historial
git checkout -b rama-nueva      # crear y cambiar a una rama
git push origin rama-nueva      # subir commits al remoto
git revert <hash>               # deshacer creando un nuevo commit
