¿Qué es un commit y por qué es importante?

Un resumen breve y listo para usar en documentación o como tooltip en tu interfaz.

Definición

Un commit es una instantánea del estado del repositorio que guarda cambios, un mensaje descriptivo y metadatos (autor, fecha, hash).

Por qué es importante
  • Historial: ver qué cambió y cuándo.
  • Reversión: volver a estados anteriores fácilmente.
  • Colaboración: revisar y combinar cambios.
  • Trazabilidad: vincular cambios a issues y despliegues.
Buenas prácticas
  • Commits pequeños y atómicos.
  • Mensajes claros en imperativo.
  • Probar antes de commitear.
  • Usar .gitignore para no subir secretos.
Prompt listo
git status
git add .
git commit -m "Mensaje claro"
git commit -am "Mensaje"
git log
git checkout -b rama-nueva
git push origin rama-nueva
git revert <hash>
